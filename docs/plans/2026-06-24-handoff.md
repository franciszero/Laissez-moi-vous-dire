## 这一段时间发生了什么（按主题）
1. **AI 批改重构（可信化）**：自由造句/翻译由本地 LLM 判分，但**模型会自信地错**。改成「人工固化规范(1-4) + 模型按规范判(5-7) + 代码确定性染色(8)」：`aigrade.py`(build_prompt/normalize_result/锚定片段逐字验证再染色)、规范在 `LLM_EXERCISES`(app.py)、开放状态集(正确/可接受变体/不自然/错/多余/拿不准)、三态人工终判、`scripts/llm_grade_bakeoff.py`+`gold/` 选模型。方法论固化进 skill `llm-graded-practice`。**LLM 是陪练不是教科书。**
2. **每个知识点都有历史**：`store.checkpoint_attempts`（机判/自评卡逐次作答）+ 卡片「📖 上次怎么答的」。AI 产出走 `ai_attempts`（更丰富，带 spans 回放）。
3. **动词变位卡**：一动词一张卡、整张范式手敲填空、逐格 `check_fr`。**答案键确定性、绝不用 LLM**：`conjugate.py` 规则生成(er/ir2/re) + **人工核对表**(être/avoir/aller/voir/pouvoir/vouloir)，`test_conjugate.py` 逐格验证。动词清单 `../L*/conjugation.json`。⚠️ **mlconjug3（用户首选）在本机坏了**（其模型要 sklearn 1.6.x、本机 1.9.0，unpickle 失败）；verbecc API 不配合——故走人工核对表。要补更多不规则：扩 `_IRREGULAR` 表 + 加测试，或修好库。
4. **product-story-critique 评审 → 引擎统一（本轮重点）**：评审指出侧栏 8 个入口本质是「练词/练卡」两套平行引擎被按内部轴切碎，"到期"碎成 4 处。用户**作废了 HANDOFF §5「等 3+ 处再抽象」**，要求消除重复开发。已做 P0/P1/P4（见下）。

## 架构最理想状态
**一个 `ReviewSession` 引擎 + 多态 `ReviewItem`。脊梁：机制统一、内容多态。**
- `ReviewItem` 接口：`id / kind / due_at / render() / grade(answer) / on_result(ok) / mastery()`；kind ∈ `word:听写/产出/理解/音/变 | checkpoint | conj | production`。
- **只写一遍**：队列/一轮/批次/导航(`ReviewSession`)、`due_today()`、一张 `attempts(item_id,kind,…)`、一张 `srs_state(item_id,kind,streak,interval,due_at)`、一个 `render(item)` 分派。
- **保留多态（本质差异，非重复）**：词的 5 维 Wilson 掌握度 + 热力图（词专有）；各 kind 的 render/grade。硬合会砍掉热力图——别。
- **不做**：跨课共享 mastery/registry（另一回事，不是本次目标）。

## 别再重推导的决策 / 坑
- 答案键(变位)**只能确定性**，绝不 LLM；AI 判分必须**对照人工固化规范**、代码管染色、人工三态终判。
- 撇号只在 `matcher.norm_fr` 判分时归一，**绝不改存储的 lemma**（DB/vocab/Anki 三处共用键）。
- 模块级辅助函数(如 `_conj_card`/`_start_cards`/`_due_checkpoint_cards`)**必须定义在其被模块级调用处之前**（侧栏入口是模块级代码）。
- 带 `key` 的控件**别条件渲染**（跨 rerun 丢状态）。
- AppTest 的 `session_state` 无 `.get()`，用属性或下标。
